# Segment 4: Split-View Magic

**Duration:** 10 seconds

---

## First Frame (Screenshot Description)

Terminal split into a 2x2 grid layout showing four different terminal sessions:

**Top-Left Pane**: Dev server running
- Shows "Server running on http://localhost:3000"
- Live request logs scrolling

**Top-Right Pane**: Test watcher
- "Watching for file changes..."
- Recent test results visible

**Bottom-Left Pane**: Git diff output
- Green lines (additions) and red lines (deletions)
- File header visible
- Syntax-highlighted code

**Bottom-Right Pane**: AI assistant session
- Claude or similar AI conversation visible
- Code suggestion displayed

Each pane has its own tab label at the top. Dividers between panes are visible but subtle.

### Key Visual Elements
- 2x2 grid layout
- Four distinct terminal sessions
- Different content in each pane
- Tab labels per pane
- Resizable dividers
- Varied terminal outputs (logs, tests, diff, chat)

---

## Video Content (10 seconds)

**0-2 sec**: Starting from single terminal view. User right-clicks on a tab.

**2-4 sec**: Context menu appears showing options:
- Duplicate
- Split Horizontal
- Split Vertical
- **2x2 Grid** (highlighted)
- Close

User clicks "2x2 Grid".

**4-6 sec**: Screen smoothly animates from single pane to four panes. Each pane populates with different content.

**6-8 sec**: All four panes show activity simultaneously:
- Server logs scroll
- Tests run
- Diff updates
- AI responds

**8-10 sec**: User drags a divider to resize panes. Text overlay: "Work on four things at once."

### Motion & Effects
- Context menu appearance animation
- Smooth split animation (0.3s transition)
- Simultaneous activity in all panes
- Divider drag animation
- Text fade-in
