# Segment 8: Persistence Demo

**Duration:** 10 seconds

---

## First Frame (Screenshot Description)

Full browser window showing Vibe Anywhere with rich state:

**Visible State:**
- Multiple tabs open (4-5 tabs in tab bar)
- Terminal with visible command history:
  ```
  $ npm install
  $ npm run build
  $ npm test
  $ git status
  ```
- Sidebar with workspace selected and highlighted
- Split view active (2 panes)
- Cursor visible in terminal

**Browser Chrome:**
- Browser URL bar showing app URL
- Mouse cursor hovering near the browser's close (X) button
- Browser tabs visible

### Key Visual Elements
- Full browser window capture
- Rich terminal history
- Multiple open tabs
- Selected workspace state
- Active split view
- Browser close button highlighted/hovered

---

## Video Content (10 seconds)

**0-1 sec**: Establish the current state - all the tabs, terminal history, workspace selection visible.

**1-2 sec**: Mouse cursor moves to browser close button (X), hovers with highlight.

**2-3 sec**: Click. Browser window closes. Screen goes to desktop/black.

**3-5 sec**: Black screen or desktop. Pause. Text overlay appears: "3 hours later..."

**5-6 sec**: Browser reopens. User types URL or clicks bookmark.

**6-8 sec**: Vibe Anywhere loads. Dashboard appears and shows EXACTLY the same state:
- Same tabs (all 4-5)
- Same workspace selected
- Same terminal content including command history
- Same split view layout

**8-10 sec**: Cursor blinks in terminal, ready. Text overlay: "Close your laptop. Come back anytime."

### Motion & Effects
- Browser close animation
- Fade to black
- "3 hours later" text overlay
- Browser open animation
- Page load (can be sped up)
- State restoration (instant, emphasize the magic)
- Text fade-in
